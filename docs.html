Overview

Daily News Briefing is an Obsidian plugin that automatically generates AI-powered news summaries on topics you care about, delivered directly to your vault as markdown notes.

6 AI Providers

Google Gemini (free!), Anthropic, OpenAI, Grok, Perplexity, OpenRouter

Auto-Scheduled

Set your time, wake up to fresh news every morning

6 Languages

English, French, German, Spanish, Italian, Chinese

Flexible Templates

5 presets + custom templates with 25 placeholders

Smart Caching

Automatic daily cache cleanup & reduced token costs

One-Click Access

Sidebar button for instant news generation

Tips for Writing Effective Prompts

  • Be specific: Clearly state what you want included and excluded
  • Use structure: Request specific formats (bullet points, tables, etc.)
  • Set priorities: Tell the AI what matters most
  • Include examples: Show the AI what good output looks like
  • Test iteratively: Start simple and refine based on results

Usage

Two ways to generate your daily news briefing:

üïê Automatic Mode

Set it and forget it - news generates at your scheduled time

üëÜ Manual Generation

Generate on-demand via command palette or sidebar button

üïê Automatic Mode (Recommended)

Once configured, the plugin automatically generates news at your scheduled time:

  1. Set your schedule time in settings (e.g., 08:00)
  2. Keep Obsidian running or open it after the scheduled time
  3. Find your briefings in the configured Archive Folder
‚ÑπÔ∏è Note: Obsidian must be running for automatic generation. If closed, the plugin will generate news when you next open Obsidian (if after the scheduled time).

üëÜ Manual Generation

Generate news on-demand using two methods:

Method 1: Command Palette

  1. Press Ctrl/Cmd + P to open command palette
  2. Type "Generate news now"
  3. Press Enter

Method 2: Sidebar Button

Click the üì∞ icon in the left sidebar.

Smart behavior:

  • If today's news exists ‚Üí Opens it
  • If today's news doesn't exist ‚Üí Creates it

üèóÔ∏è Architecture Overview

The plugin follows a modular provider-based architecture with clear separation of concerns:

Design Patterns

Pattern Implementation Purpose
Factory Pattern NewsProviderFactory Creates appropriate provider instances based on user settings
Abstract Base Classes BaseNewsProvider, BaseRetriever, BaseSummarizer Defines common interfaces for all providers
Strategy Pattern Agentic vs Search-Summarize pipelines Supports different news generation strategies
Template Pattern TemplateEngine Handles template rendering with placeholders

Pipeline Types

Agentic Pipeline (Single API Call)

Providers: Perplexity, GPT, Grok, Claude, OpenRouter

Flow: Topic ‚Üí AI Provider (handles search + summary) ‚Üí Formatted News

Pros: Simpler, fewer API calls, integrated search

Cons: Less control over search, single provider dependency

Search-Summarize Pipeline (Coordinator)

Providers: Google Search + Any Summarizer (Gemini/GPT/Grok/Claude/OpenRouter)

Flow: Topic ‚Üí Google Search ‚Üí Filter & Rank ‚Üí AI Summarizer ‚Üí Formatted News

Pros: More control, advanced search strategies, mix & match components

Cons: More API calls, requires Google setup
